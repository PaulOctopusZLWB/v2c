## 全局框架(所有页共用)

- **216px 侧栏**:Logo + ⌘K 搜索胶囊 + 主导航(今日/管道/审核/声纹/记忆/总结)+ 资料库最近日期 + ⚙ 设置 + 底部节点状态。
  - 审核/记忆项右侧的 warn 徽标 = 待办数;管道运行时「管道」项亮呼吸点。
- **46px 玻璃状态条**:左侧当前页名;右侧管道状态胶囊(运行中:呼吸点 + 阶段·目标·百分比·剩约 X 分,点击 → 管道页;空闲:「管道空闲」)+ 主题切换。
- **环境动态背景**:网格 + 极光 + 微粒(亮色隐藏)+ vignette;`prefers-reduced-motion` 全部静态。
- **全局键盘**:`1–6` 直达各页 · `⌘K` 命令面板 · `t` 明暗切换 · `Esc` 只关最上层(Dialog > ⌘K > toast)· `Enter` 执行最新 toast 主操作,首页无 toast 时 = 开始审核。输入框聚焦时字母/数字键旁路。

## 核心使用流程(验收路径)

### 1. 导入 → 转写(管道页,`2`)
1. 右栏设备卡点「导入」(或已知目录),运行控制点「运行」。
2. 左栏六阶段栈:完成 ✓ ok + mono 计数,运行中节点升级为 accent 卡(呼吸点 + n/N + 3px 进度),排队 ○ dim。
3. 中栏「实时转写」:新段从底部滑入、越旧越透明,最新行 accent 底 + 闪烁光标;头行 LIVE + 总进度(分阶段 + ETA)。
4. 右栏底部 mono 事件流:`→ 阶段` / `✕ 失败 · 目标 · 错误` / `✓ 运行完成`。失败任务在任务列表可单个/全部重试。
5. 完成:中栏出 ok 完成条「立即审核 ↵」;全局也会弹 success toast(Enter 直达审核)。勾选左栏「完成后自动跳转审核」则直接切页。

### 2. AI 预审审核(审核页,`3`)
1. 从待审队列/按天浏览打开一场会话。头部:会话名 + ✎ 重命名(↵ 保存 / esc 取消)+ mono 元信息 + 「已审 x/N」+ 160px 进度条。
2. AI 预审横幅:「N 段高置信建议直接接受;M 段可疑已前置」。`⇧A` 或点按钮批量接受高置信(有「撤销」toast 兜底),按钮变「✓ 已接受 N 段」。
3. Turn 卡:可疑卡前置,左 3px 状态边(待审=淡 warn/接受=ok/拒绝=err/存疑=warn),卡头 warn 胶囊标注原因(「置信 0.41」「说话人存疑 → 可能是 XX」「疑似幻听 · 与上下文断裂」)。
4. `j/k` 移动焦点卡(accent 描边 + 辉光),`a` 接受 / `e` 采纳建议说话人(改归属 + 接受)/ `r` 拒绝 / `f` 存疑,决策后自动前进;`space` 播放。已决策卡出右上角标并隐藏原因胶囊。
5. 其余高置信段折叠在虚线行(点击展开复核)。全部处理完出 ok 完成卡 → 「接受整场并归档」回今日。
6. 底部 mono 快捷键条 + 「剩 n 段可疑」。

### 3. 声纹指认(声纹页,`4`)
1. 星图:已识别人物 = 实心彩点,未分配 = 空心描边点;框选/点选照旧。
2. 右栏候选簇:未分配聚类显示「AI 猜测:李四 (0.71)」+「采纳 → 李四」一键归属;也可下拉手选。指认后地图即时重着色。
3. 会话身份闸机(在场/不在场确认)驱动总结放行与首页「参与人」列。

### 4. 记忆确认(记忆页,`5`)
1. 头部:「待确认记忆」+ mono 计数 + ok 胶囊「🔑 Ed25519 · did:key:…」。
2. 候选卡:类型胶囊(偏好=accent/事实=warn/承诺=紫)+ mono 日期·置信 + 正文 + 内嵌证据行(引文 + ▶ 播放 + 跳到转写)。
3. 焦点卡 `a` 确认并签名(ok 填充)/ `r` 拒绝 / `d` 搁置 / `z` 撤销上一次拒绝或搁置。
4. 确认后:卡变 ok 描边 + mono 回执「✓ memory_card.created · sig xxxx…xxxx · 已写回 40_Confirmed_Memory/」。**已签名的确认不可撤销**(事件链只增不减,这是有意的安全语义)。
5. 首页中卡「待确认记忆」整卡可点直达本页。

### 5. ⌘K 检索与命令
- `⌘K` 打开:输入即过滤;「语义检索」组(FTS5,中文子串可命中,最新在前)排在「命令」组之前,`↵` 跳转到该话语(审核页高亮);`1–6`/`t` 等命令带 mono 提示。

### 6. 危险操作确认(全局)
- 删除会话/人物、重新聚类、重新生成总结均走自研 Dialog:`esc` 取消,**`⌘↵` 才确认**(普通 Enter 不会触发破坏性操作);重命名对话框 `↵` 保存、`⇥` 采用 AI 建议(建议存在时)。

## 数据安全

- 全程**未做任何破坏性 schema 变更**:新增只有 FTS5 虚表(可随时重建)与只读 API;`memory_candidates.status` 新增 `deferred` 取值为纯增量。
- 备份工具:`src/personal_context_node/db_backup.py`(建议大版本升级前照常备份)。

## 已知的有意取舍

- 「采纳 AI 修正 `e`」目前只覆盖**说话人建议**;文本纠错(`suggested_text`)接口字段已预留,等有 ASR 备选/LLM 纠错数据源后自动出现按钮。
- 记忆确认的 `z` 不撤销已签名的确认(append-only 事件链);设计稿此处与密码学语义冲突,以安全为准。
- 输入框聚焦时 Enter 不触发 toast 主操作(与交互原型一致;README 文字与原型冲突处以原型为准)。
- FTS5 检索是词元级:中文子串完全命中;拉丁词按整词/前缀匹配,纯符号查询自动回退 LIKE 字面匹配。
