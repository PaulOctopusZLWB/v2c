PCN · WEB 控制台 · 前端设计调研

v2c 前端系统性优化调研

目标:科技感 · AI 原生 · 操作流顺畅。基于对 web/ 全部源码(theme.css / styles.css / App.tsx / 32 个 feature 组件)与 README 流程的阅读。

01 · 现状:基础比预期扎实

这不是一个需要推倒重来的前端。已有一套自洽的 "Cold Slate" 设计系统,很多优化可以在它之上做加法:

✓ 完整 token 体系
8pt 间距、8 级字阶、z-index 分层图、明暗双主题、等宽数字 — 设计纪律很好。
✓ 单一强调色原则
电青色 accent 只用于「活跃/焦点/进行中」,语义状态色独立 — 这是做科技感的正确底子。
✓ 高效交互已起步
⌘K 命令面板、j/k/a/r/f 键盘审核、骨架屏、空状态引导、reduced-motion 兜底。
✓ 流水线心智模型
设备→导入→转写→审核→观点→发布 六阶段 spine 已存在,是全局叙事的锚点。

02 · 差距分析(对照三个目标)

目标 A 科技感 — 视觉在,动效缺席
  • 全局动效只有 4 个 keyframes(pulse / slidein / spin / shimmer)+ 0.12s 的通用 transition。没有页面切换过渡、没有数据入场编排、没有微交互反馈。
  • 波形组件 .wave 是几根静态 div 条 — 一个音频为核心的产品,最该有生命力的元素是死的。
  • VoiceprintMap(声纹投影图,3.2 万行里最大的可视化)是全应用最有科技感潜力的画面,目前是普通 canvas 散点,可升级为发光粒子 / 聚类动画 / 平滑缩放。
  • 进度条、任务状态是平铺文本+细条,缺少「机器在为我工作」的实时感。
目标 B AI 原生 — AI 在后台,不在界面里
  • 无流式呈现:转写、摘要、观点都是「跑完→整块出现」。AI 原生产品的标志性体验(token 级流式输出、逐段点亮)完全缺失 — 需要后端加 SSE。
  • AI 不给建议:审核是纯人工逐段过;LLM 明明有置信度/情绪/说话人信息,却没有「AI 预审:这 12 段可疑,其余建议一键接受」这类协作式交互。
  • 确认流断裂:观点/记忆候选在 UI 里只读,确认要切去 Obsidian 改 checkbox 再跑 CLI — 最核心的价值环节离开了产品。
  • ⌘K 面板已是很好的骨架,可进化为自然语言入口(「昨天下午和谁聊了报销?」)— 需要后端检索/LLM 接口。
目标 C 操作流顺畅 — 好流程被粗糙节点打断
  • 重命名/删除会话用的是 window.prompt / window.confirm — 系统原生弹窗与整体质感严重断裂,是最扎眼的粗糙点。
  • 导入→处理→审核 靠轮询刷新 + 手动切 tab;处理完成后没有「去审核」的自动引导,用户要自己发现状态变了。
  • 键盘审核流很好,但缺进度反馈:没有「剩余 x/y」、没有会话审完的完成时刻,长会话审核像走不完的隧道。
  • 首次使用只有空状态文案,没有 onboarding 路径把「接设备→导入→等转写→审核」串成一次引导。

03 · 优化路线(按依赖分三层)

层 1
纯前端
无需后端改动
统一动效系统(入场编排、tab 过渡、微交互、状态变化动画)· 自研 Dialog 替换原生 prompt/confirm · 实时 canvas 波形与播放动画 · 审核进度条 + 完成时刻 · 声纹图视觉升级(发光、缓动缩放)· 管道运行时的「控制室」实时感。
层 2
流程重构
前端为主,少量 API
导入→审核的自动引导(完成 toast 直达)· AI 预审模式:按置信度排序、可疑段前置、批量建议 · 观点/记忆候选在 UI 内直接确认(替代去 Obsidian 改 checkbox)· 首次使用 onboarding。
层 3
需后端配合
SSE / 新接口
流式转写与摘要(SSE 推送,前端逐段点亮)· AI 预审接口(段级可疑度)· 自然语言检索进 ⌘K · 记忆确认 API(签名事件直接从 UI 触发)。

04 · 建议的切入点

性价比最高的第一步是 层 1 的「动效系统 + 控制室实时感」 加上 层 2 的「AI 预审审核流」:前者直接兑现「科技感」,后者同时兑现「AI 原生」和「操作流顺畅」,且都能先用高保真原型验证,再落回 React 代码。下方回答几个问题,我就开始出设计稿。
